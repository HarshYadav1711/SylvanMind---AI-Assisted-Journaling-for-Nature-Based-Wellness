# Demo screenshot placeholders

Add PNG screenshots to this folder so the main README can display them. Use the filenames below. Do not commit large or unrelated images.

| Filename | Description | How to capture |
|----------|-------------|----------------|
| **editor.png** | **Journal editor page** — The Journal view showing the text area for writing, the ambience selector (forest / ocean / mountain), and the Save / Analyze actions. Optionally include the user ID field and the list of recent entries below the editor. | Run the app, open the Journal page, optionally enter sample text and select an ambience. Capture the full editor section (and list if visible). |
| **analysis.png** | **Journal analysis results** — The same Journal page after the user has clicked “Analyze”: the displayed emotion, keywords, and summary for the current journal text. | After running analysis on the Journal page, capture the results panel (emotion, keywords, summary). |
| **insights.png** | **Insights dashboard** — The Insights page showing aggregated stats: total entries, top emotion, most used ambience, and recent keywords for the current user. | Run the app, open the Insights page (with a user that has some entries and analysis). Capture the dashboard. |

Place the three files in the repo as:

- `docs/editor.png`
- `docs/analysis.png`
- `docs/insights.png`

The README references these paths; once the files exist, the images will render in the Demo Screenshots section.

**Animated demo:** For a short walkthrough GIF, add `docs/demo.gif` showing (1) writing a journal entry, (2) running analysis, (3) viewing the Insights page. See the README **Demo** section for instructions.
